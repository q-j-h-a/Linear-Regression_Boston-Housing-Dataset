优化算法：梯度下降与参数迭代

在定义了均方误差（MSE）这把量测“预测误差”的尺子后，我们的核心目标变得非常明确：寻找一组最优的特征权重矩阵 $\mathbf{w}$，使得这把尺子量出的误差降到最低。这个不断寻找最优参数的过程，在机器学习中被称为优化（Optimization）。

1. 梯度下降 (Gradient Descent)：蒙眼下山的艺术

正如我们在前置知识中提到的，你可以把寻找最优参数的过程，想象成你被蒙上双眼放在了一座高低不平的山坡上（这座山就是损失函数构成的误差曲面）。你要怎么走到谷底？

最稳妥的策略是：用脚探一探四周，找出最陡峭的下坡方向，然后往那个方向迈出一步。不断重复这个过程，直到不管往哪边迈都是上坡，你就知道自己到达谷底了。在微积分中，梯度（Gradient）指向的是函数值上升最快的方向。因此，我们的参数必须沿着梯度的反方向进行更新。

通过对均方误差 MSE 函数求偏导，我们可以得到参数矩阵的梯度向量 $\nabla_{\mathbf{w}} J$：

$$ \nabla_{\mathbf{w}} J = \frac{1}{m} \mathbf{X}^T (\mathbf{\hat{y}} - \mathbf{y}) $$
💡 维度核对：为什么 $\mathbf{X}$ 必须要转置？
这并不是随意的数学推导，而是为了满足严谨的矩阵运算规则。回忆一下：误差向量 $(\mathbf{\hat{y}} - \mathbf{y})$ 的维度是 $(m, 1)$。而我们希望求得的梯度向量 $\nabla_{\mathbf{w}} J$，其维度必须和权重 $\mathbf{w}$ 完全一致，即 $(d+1, 1)$。
由于原始特征矩阵 $\mathbf{X}$ 的维度是 $(m, d+1)$，只有将其转置为 $\mathbf{X}^T$ $(d+1, m)$，再与误差向量相乘：$(d+1, m) \times (m, 1)$，最终才能完美得到维度为 $(d+1, 1)$ 的梯度向量。

2. 参数更新与学习率 (Learning Rate)

算出了梯度（下山的方向）后，我们还要决定每次迈出的“步子”有多大。这个步长由一个核心的超参数——学习率（通常记作 $\alpha$）来控制。参数的数值迭代更新公式如下：

$$ \mathbf{w} := \mathbf{w} - \alpha \nabla_{\mathbf{w}} J $$
⚠️ 工程避坑指南：学习率的抉择
• 步子迈得太大 ($\alpha$ 过大)： 你可能会直接跨过谷底，跑到对面的半山腰上，甚至越走越高。在代码中的表现是：Loss 值不降反升，最后变成 NaN（数值溢出爆炸）。
• 步子迈得太碎 ($\alpha$ 过小)： 虽然很稳，但下山速度极其缓慢，你的模型可能需要训练几万次（Epoch）才能走到谷底。
在后续的实验中，通常建议从 0.01 或 0.001 开始尝试调整你的学习率。

3. 迭代训练：Epoch 的概念与可视化

上述“计算预测值 -> 计算梯度 -> 更新参数”的完整流程，我们称之为一次迭代（Iteration）或一个轮次（Epoch）。随着 Epoch 的增加，我们可以监控损失函数的值，正常的模型训练应该呈现出一条平滑下降的 Loss 曲线：

4. 核心代码实现：反向传播

借助 NumPy，我们将上面复杂的公式浓缩成了短短几行极其优雅的代码：

import numpy as np

def gradient_descent_step(X, y, w, learning_rate):
    """
    执行一步梯度下降，更新参数矩阵
    """
    m = len(y)
    
    # 1. 前向传播：计算当前的预测值和误差
    y_pred = np.dot(X, w)
    error = y_pred - y
    
    # 2. 反向传播：计算梯度 (注意 X 的转置)
    # 公式: (1/m) * X^T * (y_pred - y)
    gradient = (1 / m) * np.dot(X.T, error)
    
    # 3. 参数更新：向着梯度的反方向迈步
    w_new = w - learning_rate * gradient
    
    return w_new
