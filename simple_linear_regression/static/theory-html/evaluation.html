模型评估：度量泛化能力与预测置信度

通过梯度下降，我们的模型在训练数据上找到了使 Loss 最小的参数 $\mathbf{w}$。但这就能说明模型很好吗？不一定。一个只会死记硬背的差学生，在做做过的模拟题（训练集）时也能拿满分，但一遇到新题（测试集）就原形毕露。

因此，我们需要用测试集（Test Set）的数据来评估模型对未知数据的适应能力，这在机器学习中被称为泛化能力（Generalization）。

1. 回归任务的核心评价指标

在训练时我们使用了均方误差（MSE），但它的单位是“房价的平方”，在业务上难以向客户解释。因此，在评估阶段，我们通常使用以下两个指标：

(1) 均方根误差 (Root Mean Squared Error, RMSE)

RMSE 就是将 MSE 进行开方。这样做的好处是让误差的单位与原始房价（千美元）保持一致，使其具备直观的业务和经济学含义。

$$ RMSE = \sqrt{\frac{1}{m} \sum_{i=1}^{m} \left( \hat{y}^{(i)} - y^{(i)} \right)^2} $$

(2) 决定系数 ($R^2$ Score)

RMSE 依然是一个带有量纲的绝对数值（比如 RMSE = 4.5），如果不了解波士顿房价的基准，我们很难判断这个误差是好是坏。为此，统计学引入了决定系数 $R^2$。它表示模型能够解释目标变量方差的百分比：

$$ R^2 = 1 - \frac{\sum_{i=1}^{m} \left( \hat{y}^{(i)} - y^{(i)} \right)^2}{\sum_{i=1}^{m} \left( y^{(i)} - \bar{y} \right)^2} $$

注：公式分母部分代表了如果用所有房价的平均值 $\bar{y}$ 去做基准预测时的总误差。

⚠️ 学术避坑指南：$R^2$ 可以是负数吗？
很多初学者误以为 $R^2$ 是某个值的平方，所以范围一定是 $[0, 1]$。这是错的！
当模型表现优异时，$R^2$ 趋近于 1；当模型预测的准确度和直接“猜所有房子的价格都是均价”一样时，$R^2 = 0$；而当模型学到了完全错误的规律（比如面积越大房价越便宜）时，$R^2$ 完全可以是一个负数。

3. 代码实现：计算评估指标

在 Python 中，借助于 NumPy 依然可以极其高效地计算这两个核心评估指标。你可以在你的项目中加入以下逻辑：

import numpy as np

def evaluate_model(y_true, y_pred):
    """
    计算线性回归的评估指标 RMSE 和 R-squared
    """
    # 确保维度匹配
    y_true = y_true.flatten()
    y_pred = y_pred.flatten()
    m = len(y_true)
    
    # 1. 计算均方根误差 (RMSE)
    mse = np.sum((y_true - y_pred) ** 2) / m
    rmse = np.sqrt(mse)
    
    # 2. 计算决定系数 (R-squared)
    y_mean = np.mean(y_true)
    # SSR: 残差平方和 (Sum of Squared Residuals)
    ssr = np.sum((y_true - y_pred) ** 2)
    # SST: 总平方和 (Total Sum of Squares)
    sst = np.sum((y_true - y_mean) ** 2)
    
    r2_score = 1 - (ssr / sst)
    
    return rmse, r2_score
